Requirements Index
==================

Purpose
-------

Use this index when you need acceptance rules, requirement IDs, or traceability between docs, tests, and service contracts.

Artifacts
---------

- Functional requirements -> `docs/requirements/functional/`
- Non-functional requirements -> `docs/requirements/non-functional/`
- Traceability matrix -> `docs/requirements/traceability.md`
- Per-service OpenAPI contracts -> `specs/<service>/openapi.yaml`
